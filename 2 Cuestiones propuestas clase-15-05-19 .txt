1. ¿Que te proponias con la creacion del branch?

Para llevar un mejor control del código. 

Se crean bifurcaciones "branchs" del estado del código que crea un nuevo camino de cara a la evolución del código, 
en paralelo a otras ramas que se puedan generar.

La rama "master" es la rama por defecto cuando creas un repositorio. 

Puedo crear nuevas ramas durante el desarrollo y fusiónalas a la rama principal cuando termines.
git checkout -b A1
git push origin A1

git checkout master
git merge A1
